# AGENTS.md

## プロジェクト概要

このリポジトリは、Flowist の一時的なティザーLPを管理するためのものです。

Flowistは、iPhoneで録音した音声を日本語で文字起こしし、オンデバイスAIで要約・タイトル生成まで行うローカルAIメモアプリです。

このリポジトリは、Flowist本体のiOSアプリリポジトリではありません。  
このページは一時的な紹介用ページであり、将来的に削除される可能性があります。

## 目的

Flowistのコンセプトを、少人数に素早く見せるためのシンプルな紹介ページを作ること。

このページで伝えたい中心メッセージは以下です。

> ユーザーがAIに話しかける前に、もう整理されている。

これは正式なマーケティングサイトではありません。  
作り込みすぎないでください。

## プロダクト思想

Flowist という名前は、人間をあるべきシーンでフロー状態に乗せる仕組み、という意味を持つ。

Flowist は、会議メモをできるだけ精密に文字起こしするためのアプリではない。  
会議中にタイピングしたり手書きメモを取ったりすることを減らし、参加者を会議そのものに集中させるための仕組みである。

会議中に記録作業へ没入しても、本質的な業務価値は大きくならない。  
本来集中すべきなのは、聞くこと、考えること、話すこと、判断すること、関係性を作ることである。

AI によるフルオート化や完全性を求めることと、業務を助けることは目的が違う。  
Flowist は完全自動の議事録作成ツールではなく、会議中のメモ取り負荷を減らし、会議後に人が手直しして共有できる「下書き」を生成するローカル AI メモアプリとして改善を進める。

そのため、今後の改善では「高精度な完全議事録」を目指すのではなく、以下を重視する。

- 録音前後のコンテキスト付与
- 断定しすぎない要約生成
- 人が補正しやすい編集体験
- 利用前の注意喚起
- 共有前の確認
- ローカル処理アプリとしての責任範囲の明確化

## 基本方針

## 文言方針

日本語で書いてください。

トーン：

- シンプル
- 落ち着いている
- わかりやすい
- 売り込みすぎない
- 技術に寄りすぎない

使ってよい表現：

- 「ユーザーがAIに話しかける前に、もう整理されている。」
- 「iPhoneで録音した音声を、文字起こし・要約・タイトル生成まで行うローカルAIメモアプリ」
- 「外部LLM APIを前提にしない」
- 「Closed Beta準備中」

避ける表現：

- 「完全プライベート」
- 「絶対安全」
- 「完璧な要約」
- 「すべて自動化」
- 「あらゆる会議に対応」
- 「仕事が爆速になる」

プライバシーについて書く場合は、以下のように表現してください。

- 「外部LLM APIを前提にしない設計」

以下のように断定しないでください。

- 「一切外部送信しない」

ただし、実装とプラットフォーム挙動が明確に確認できている場合は除きます。

## デザイン方針

- ダークテーマ（暗い背景）
- ネオン系のアクセント（シアン / パープル / ピンクのグラデーション）とグロー
- グラスモーフィズム調の角丸カード
- 情報量は絞り、要素は詰め込みすぎない
- 大きめのヒーローコピー
- モバイルファースト
- 重いアニメーション（`@keyframes`）は使わない。波形・オービット等の装飾はすべて静的に表現する
- 雰囲気づくりのネオン装飾画像は使ってよい（プロダクトと無関係な多用は避ける）

正式な企業サイトではなく、軽いティザーとして見えることを優先してください。

## 開発コマンド

以下のコマンドで動作することを期待します。

bash npm install npm run dev npm run build

ローカル実行に追加サービスを必要としないでください。

## 完了条件

- npm install が成功する
- npm run dev でローカル開発サーバーが起動する
- npm run build が成功する
- iPhoneサイズの画面でも破綻しない
- 実装が小さく読みやすい
- Vercelにそのままインポートできる
- 不要な依存ライブラリが追加されていない
- バックエンド、認証、CMS、Analytics、フォーム機能が追加されていない

## レビュー観点

完了前に以下を確認してください。

- これはまだ一時的なティザーLPか
- 不要な依存ライブラリを追加していないか
- AIやプライバシーについて誇大表現をしていないか
- 10秒以内にFlowistが何のアプリかわかるか
- モバイルで見やすいか
- 特別な設定なしでVercelにデプロイできるか
