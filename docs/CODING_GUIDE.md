# Coding Guide

このリポジトリは、Flowist の一時的なティザーLPを小さく読みやすく保つためのものです。実装は Vite + React + TypeScript + CSS を前提にします。

## 基本方針

- 1ページのティザーLPとして実装する
- 小さく、明快で、捨てやすいコードにする
- 依存ライブラリを増やさない
- バックエンド/API Routesを追加しない
- 複雑な状態管理やルーティングを入れない
- 文章は日本語で、売り込みすぎない

## React / TypeScript

- コンポーネントは必要になったときだけ分ける
- props は複雑にしない
- 汎用化より、読みやすさを優先する
- 型は明示しすぎず、推論で十分なところは推論に任せる
- データ量が少ない場合は、同じファイル内の配列定義でよい

## CSS

- 通常のCSS、またはCSS Modulesを使う
- レイアウトはモバイルファーストで書く
- 余白、文字サイズ、幅の制限を優先して読みやすくする
- 重いアニメーションや装飾を入れない
- ストック画像に頼らない

## 文言

使ってよい表現:

- 「ユーザーがAIに話しかける前に、もう整理されている。」
- 「iPhoneで録音した音声を、文字起こし・要約・タイトル生成まで行うローカルAIメモアプリ」
- 「外部LLM APIを前提にしない」
- 「Closed Beta準備中」

避ける表現:

- 「完全プライベート」
- 「絶対安全」
- 「完璧な要約」
- 「すべて自動化」
- 「あらゆる会議に対応」
- 「仕事が爆速になる」

## 完了前の確認

- 10秒以内に Flowist が何のアプリかわかるか
- iPhoneサイズでも読みやすいか
- 不要な依存ライブラリを追加していないか
- 誇大なAI表現やプライバシー表現がないか
- `npm run build` が成功するか
